Missing `role="group"` on filter containers and `aria-pressed` on stateful toggle buttons makes keyboard and screen reader navigation unclear.\n**Action:** Always ensure that custom button groups acting as tabs or filters have an appropriate `role="group"` on their container, and use `aria-pressed` on the buttons themselves to indicate the active state. Add `aria-label` to search inputs lacking visible labels.
## 2026-07-10 - [Add keyboard accessibility to file dropzone]
**Learning:** Custom interactive elements like file dropzones need explicit keyboard support (e.g., tabindex="0", keydown listeners) and focus management when masking native HTML inputs.
**Action:** Always ensure tabindex, aria-label, role, and keydown listeners are added for keyboard accessibility when building custom UI elements that hide native form controls.
